## Goal

Establish a repeatable, low-friction release process:
- Feature branches → merge to main → accumulate commits → push tag → GitHub Release created automatically
- `CHANGELOG.md` generated by git-cliff from conventional commits (replaces manual file)
- `install.sh` gains VERSION pinning support (unblocks TODO #6, #17, #25)

---

## Decisions

| Question | Decision |
|----------|----------|
| Who initiates release? | Developer pushes tag (Option A) |
| Changelog generation | git-cliff generates CHANGELOG.md fully (A1) |
| History migration | Full replacement — git-cliff regenerates from v1.0.0 (M1) |
| Release assets | None yet — raw GitHub URLs; assets added in TODO #17 |
| macOS CI | `macos-latest` (repo is public → free) |

### `cliff.toml`

```toml
[changelog]
header = """
# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

---
"""
body = """
## [{{ version | trim_start_matches(pat="v") }}] - {{ timestamp | date(format="%Y-%m-%d") }}
{% for group, commits in commits | group_by(attribute="group") %}\
### {{ group }}
{% for commit in commits %}\
- {{ commit.message | upper_first }}
{% endfor %}
{% endfor %}\
"""
trim = true

[git]
conventional_commits = true
filter_unconventional = true
commit_parsers = [
  { message = "^feat",     group = "Added"        },
  { message = "^fix",      group = "Fixed"         },
  { message = "^refactor", group = "Changed"       },
  { message = "^docs",     group = "Documentation" },
  { message = "^test",     skip = true             },
  { message = "^chore",    skip = true             },
  { message = "^plan",     skip = true             },
  { message = "^revert",   skip = true             },
]
```

### `.github/workflows/ci.yml`

Triggers: pull_request, push to main.

- `test` job — `macos-latest`: `brew install bats-core` → `bats proxied-claude.bats`
- `lint` job — `ubuntu-latest`: `shellcheck proxied-claude claude-proxy install.sh`

### `.github/workflows/release.yml`

Triggers: `push: tags: ['v*']`

- `ubuntu-latest`, `permissions: contents: write`
- Full checkout (`fetch-depth: 0`)
- Install git-cliff from GitHub releases
- `git cliff --current --strip header > RELEASE_NOTES.md`
- `gh release create "$GITHUB_REF_NAME" --title "$GITHUB_REF_NAME" --notes-file RELEASE_NOTES.md`

### `claude-proxy` — VERSION constant

Updated manually before each release tag. No automation — developer edits `VERSION="x.y.z"` as part of the release commit.

### `CHANGELOG.md`

Replaced entirely. Migration: run `git cliff -o CHANGELOG.md` to regenerate from full history (v1.0.0 onwards). Output will be shorter than the current manual file — accepted tradeoff.

## Dev Release Workflow

```bash
# 1. Ensure main is clean and up to date
git checkout main && git pull

# 2. Update VERSION in claude-proxy (one line: VERSION="x.y.z")

# 3. Regenerate CHANGELOG.md
git cliff --tag vX.Y.Z -o CHANGELOG.md

# 4. Commit and tag
git add CHANGELOG.md claude-proxy
git commit -m "chore: release vX.Y.Z"
git tag vX.Y.Z
git push origin main --tags
# → CI catches the tag, creates GitHub Release automatically
```
